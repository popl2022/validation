Some of our tests derive from other cycles: they basically are variations on the SB, MP and R tests above, plus one isolated write. By transitivity of the modification order those tests are redundant when considered as characterisations of SC violations. Nevertheless, they remain useful for hardware experiments, where varying testing context may prove beneficial.

Finally, tests that operate on a single variable are also part of our test base, as well as a few tests for testing the fence effect of the (locked) compare-exchange instruction.

Each test has run at least 5 × 108 times (more than 1.1 109 times on average).

Tested machines are:

  • CoreI5-dual, a dual-core (Intel Core i5-530005U; 2.30 Ghz) HP EliteBook laptop, running Linux 4.4.0.
  • CoreI5, a quad-core (Intel Core i5-10310U; 1.70GHz) Dell Latitude 7410 Laptop, running Linux 5.0.8.
  • CoreI7, a quad-core (Intel Core i7-8665U; 1.90Ghz) Lenovo ThinkPad X - X390 Yoga laptop, running Linux 5.4.0.
  • Chianti an octo-core (2x quad-core Intel Xeon E5620; 2.4 GHz) HP Z800 precision desktop, running Linux 3.13.0.
  • Limoux, a 12-core (2x hexa-core Intel Xeon E5-2620; 2.4 GHz) Dell Precision 7810 desktop, running Linux 4.9.0.
  • Teraram, a 40-core (4x 10-core Intel Xeon E7-4870; 2.40 GHz) Dell PowerEdge R910 server, running Linux 4.15-0.
  • Estephe, an octo-core (octo-core Intel Xeon W-2145 CPU; 3.70GHz), Dell Precision 5820 desktop, running Linux 5.4.0.
  • Barsac, a 20 core (2x 10-core Intel Xeon CPU E5-2640; 2.40GHz) server, running Linux 4.15.0.

The older chianti does not have access to the WT (Write-Through) memory type and only the most recent coreI5, coreI7 and estephe implement the clflushopt instruction.
